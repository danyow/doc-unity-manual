Fixing lightmap UV overlap

Each lightmap contains a number of charts . At run time, Unity maps these charts onto mesh faces, and uses the chartsâ€™ lighting data to calculate the final appearance. Because of the way GPU sampling works, data from one chart can bleed onto another if they are too close to each other. This usually leads to unintended artifacts such as aliasing, pixelation, and so on.
