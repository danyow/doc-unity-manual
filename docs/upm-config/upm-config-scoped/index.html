范围注册表身份验证

一些组织将他们自己的包托管在需要身份验证才能访问的私有包注册表上。如果您是这类组织的员工或客户，则必须使用 npm 身份验证来配置一个范围注册表。要进行设置，请获取 npm 身份验证令牌，然后将该令牌添加到您的用户配置文件中。

获取 npm 身份验证令牌​

对于各注册表提供程序，创建和访问 npm 身份验证令牌的过程均不同。例如，JFrog 的 Bintray 和 Artifactory 代码仓库管理器使用不同的过程从 npm 生成身份验证令牌。这是典型过程的示例，但您需要遵循特定包注册表提供程序为您的范围注册表推荐的过程。

要从 npm 获取身份验证令牌：

1.在您的机器上本地安装 npm。

2.从终端输入以下命令以登录注册表：

`$ npm login --registry &lt;registry url&gt;`

3.找到并打开生成的 .npmrc 文件。

4.找到 _authToken 或 _auth 条目并复制其值（请参见下面的示例）。

根据注册表，令牌字符串可以是 GUID、令牌或专有格式的字符串。

.npmrc 文件示例​

这是一个包含 _authToken 属性的 .npmrc 文件示例：

registry=https://example.com:1234/mylocation/
//example.com:1234/mylocation/:_authToken=&lt;AUTH TOKEN&gt;

这是一个包含 _auth 属性的 .npmrc 文件示例：

registry=https://example.com:1234/mylocation
_auth=&lt;AUTH TOKEN&gt;
email=&lt;EMAIL&gt;
always-auth=true

配置身份验证信息​

使用 npmAuth 配置模式，将需要身份验证的每个范围注册表的令牌信息存储在 .upmconfig.toml 用户配置文件。将此信息保存到配置文件后，Package Manager 将针对向文件中的每个注册表发出的每个请求提供您的身份验证信息。

按照以下说明将您的身份验证信息添加到用户配置文件中：

1.找到 .upmconfig.toml 用户配置文件。如果该文件尚不存在，请创建一个空文本文件。

2.使用一种模式格式化您的身份验证信息，具体取决于您使用的是 Bearer（基于令牌）还是 Basic（Base64 加密）身份验证机制：

[npmAuth."&lt;REGISTRY URL&gt;"]
&lt;TOKEN-PROPERTY&gt; = "&lt;TOKEN-VALUE&gt;"
email = "&lt;EMAIL&gt;"
alwaysAuth = &lt;BOOLEAN&gt;

下表阐明了如何指定配置文件值：

条目描述：
[npmAuth."&lt;REGISTRY URL&gt;"]必需。注册表 URL。例如，[npmAuth."https://example.com:8081/mylocation"]。
&lt;TOKEN-PROPERTY&gt; = "&lt;TOKEN-VALUE&gt;"必需。从 npm 注册表生成的身份验证令牌。可以是 GUID、令牌或专有格式的字符串。例如，可以是 token = "&lt;AUTH TOKEN&gt;" (Bearer) 或 _auth = "&lt;BASE64 TOKEN&gt;" (Basic)。
email可选。与注册表中用户电子邮件匹配的用户的电子邮件地址。
alwaysAuth可选。如果包元数据和 tarball 不在同一服务器上，则设置为 true。通常，您可以从生成的 .npmrc 文件复制该值。
