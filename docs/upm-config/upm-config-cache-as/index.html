Customize the Asset Store cache location

The Package Manager maintains a cache for packages that you get from the Asset Store. The Package Manager stores this cache in a default location, which you can override.

You might want to override the Asset Store cache location for several reasons:

  • You want to save space on your internal drive.
  • You want to store the cache on a shared drive, which is accessible to others.
  • You want to store the cache in a folder that’s not in a system directory.

By default, the Package Manager uses the following folder structure for its Asset Store package cache:

&lt;asset-store-cache-root&gt;
  └── Asset Store-5.x
      └── &lt;subfolders for Asset Store vendors&gt;

For information on the default location of the Asset Store package cache root, see Location of downloaded asset files.

Note : The Package Manager maintains separate caches for registry packages and Asset Store packages. For information on overriding the default location for the global cache, see Customize the global cache location.

To override the default location of the Package Manager’s cache for Asset Store packages, you can use the following methods:

  • The Preferences window is the recommended method because of its ease and permanence. When you change the package cache location using this method, Unity stores the preference for the current session and future sessions.
  • The ASSETSTORE_CACHE_PATH environment variable. This method isn’t permanent unless you use Advanced System Settings (Windows) or a .profile or .bash_profile file (macOS or Linux).

先决条件​

If you use the environment variable method, you should have a basic familiarity with:

  • Command line (Windows) or bash shell (macOS or Linux).
  • Creating or editing .profile or .bash_profile command line profiles (macOS or Linux).

Important notes before you begin​

  • Regardless of the method you choose, changing the cache location tells the Package Manager to use that location going forward. Existing packages that Package Manager stored in the original location remain in that folder structure.
  • If you used the environment variable method to change the cache location, then you can’t use the Preferences window to change the location.

Using the Preferences window​

To use the Preferences window to override the default location of the Asset Store package cache, follow these steps.

  1. Use one of the following methods to open the Preferences window:

    • Open the Unity menu (macOS) or the Edit menu (Windows and Linux) and select Preferences .
    • Open the Package Manager window, click the settings icon , and select Preferences .

  2. Select the Package Manager category.

  3. Under My Assets open the menu beside Cache Location .

  4. Choose Change Location .

  5. Choose a new location for the Asset Store package cache.

The Preferences window with the Package Manager category selected

Using the environment variable​

Configure the ASSETSTORE_CACHE_PATH environment variable to override the default location of the Asset Store package cache.

Windows (temporary)​

You must configure the ASSETSTORE_CACHE_PATH environment variable every time you use Unity by following these steps:

  1. Close the Unity Editor and Hub if they’re already running.
  2. From a command prompt, use the set command to configure the ASSETSTORE_CACHE_PATH environment variable.
  3. Launch the Hub from the same command prompt session.

The following example uses the Windows command prompt to override the location of the Asset Store package cache:

set ASSETSTORE_CACHE_PATH=%ALLUSERSPROFILE%\Unity\asset-cache
echo "Launching Unity with this location for Asset Store packages: '%ASSETSTORE_CACHE_PATH%'" 
"C:\Program Files\Unity Hub\Unity Hub.exe"

Windows (permanent)​

To make the ASSETSTORE_CACHE_PATH environment variable configuration permanent, you must use Advanced System Settings in Windows:

  1. Open the Advanced System Settings dialog.
  2. Select the Advanced tab.
  3. Click Environment Variables .
  4. Click New .
  5. Create an environment variable whose name is ASSETSTORE_CACHE_PATH. Set the value to the path of the cache location.

macOS and Linux (temporary)​

You must configure the ASSETSTORE_CACHE_PATH environment variable every time you use Unity by following these steps:

  1. Close the Unity Editor and Hub if they’re already running.
  2. From a terminal application, use the export command to configure the ASSETSTORE_CACHE_PATH environment variable.
  3. Launch the Hub from the same terminal session.

The following example uses a terminal session to override the location of the Asset Store package cache:

export ASSETSTORE_CACHE_PATH=/dev/ssd/shared/Unity/asset-cache
echo "Launching Unity with this location for Asset Store packages: '$ASSETSTORE_CACHE_PATH'"
"/Applications/Unity Hub.app/Contents/MacOS/Unity Hub"

macOS and Linux (permanent)​

To make the ASSETSTORE_CACHE_PATH environment variable configuration permanent, you must use a .profile or .bash_profile file.

  1. Open your .profile or .bash_profile file, or create one if it doesn’t exist.
  2. Add an export command to the bottom of the file. Refer to the macOS and Linux example above.
  3. Save the file and close your terminal window.
  4. Apply the new profile by restarting the terminal window.
