发布到 Asset Store

将资源发布到 Asset Store 的基本工作流程如下：

1.创建发布者帐户。 2.创建新的资源包草案。还可以随时删除草案，然后新建草案。 3.将资源上传到资源包。 4.填写资源包详情。 5.提交资源包以供批准。 6.查看您的资源的 Asset Store 提交状态。

在等待 Unity 是否已接受将资源包发布到 Asset Store 的通知时，可以设置付款资料，以便可以获取资源包带来的收入。如果尚未如此操作，还可以为您的发布者帐户设置客户支持信息。

将资源包发布到 Asset Store 后，可以执行以下操作来提高资源包被发现的可能性并鼓励客户购买您的资源包：

  • 向资源包添加标签，便于客户找到资源包。
  • 将帐户关联到 Google Analytics 以跟踪您在 Asset Store 上的资源包页面的网站流量。
  • Unity 可能会为您提供机会将资源包加入到 Asset Store 促销中。您可以参与 Unity 的促销或者创建自己的促销。
  • 授权 Unity 就您的资源包向您的客户退款。

如果要更新或增强现有资源包，可以创建主要升级版本。将升级发布到 Asset Store 后，可以弃用旧资源包，以便新用户只能找到最新版本。还可以创建一个“精简”升级版本，该版本可用于让客户试用资源包，而无需付全款。

在 Publisher Account 页面上的选项卡中可以执行以下操作：

  • 阅读关于资源包的评论 (Reviews)。
  • 检查销量 (Sales) 和收入 (Revenue)。
  • 查看资源包的下载量 (Downloads) 以及您发放给客户的任何兑换券 (Vouchers) 的相关统计信息。
  • 管理您团队中的用户 (Users)。
  • 管理您的帐户信息 (Info)。

如需上述任何过程的其他帮助信息，请参阅在线的 Asset Store 常见问题解答和 Asset Store 论坛。资源已被 Asset Store 接受的发布者还可以申请加入发布者论坛。

概念​

以下是您需要了解的关于“发布到 Unity Asset Store”的一些重要概念：

概念：定义：
Unity ID（帐户）您在 Unity 中使用的用户帐户。Unity ID 可以用于购买订阅、管理项目并与团队协作以及在 Asset Store 上获取工具和样本。

仅在使用 Unity 帐户登录后，才能访问和管理您在 Asset Store 中的购买内容。如果您决定创建发布者帐户，也需要 Unity 帐户。
发布者帐户该帐户与您的 Unity 帐户绑定，但还可用于创建和发布资源包、设置付款信息、查看销售额和收入以及管理您在 Asset Store 中的个人资料的各个方面。
付款资料在付款资料中可以指定 Unity 为资源销售进行付款的方式（和频度），以及 Unity 是否需要预扣税（按某些税务机关的要求）。付款资料与组织相关联。您必须是该组织的所有者才能查看、创建或更新付款资料。有关更多信息，请参阅管理您的组织。
Asset Store Tools必须使用这些工具才能从 Unity 内部创建要在 Asset Store 中出售的资源包。可以从 Asset Store 网站本身获取这些工具，就像 Asset Store 中提供的任何其他资源一样。请将工具下载到计算机并导入到项目中。

Asset Store Tools 资源包中还包含验证器 (Validator) 工具。在上传和提交 Asset Store 资源包以供审查之前，可以使用此工具来检查 Asset Store 资源包。在您将资源提交到 Asset Store 时，Asset Store 策划团队也会使用此工具来验证资源包，这是资源审查过程的必要操作，因此您对资源包运行此工具可以缩短审查时间。但是，请注意，验证器工具无法完全模拟审批过程，有时可能会产生误报。
资源包状态描述每个资源包位于生命周期的哪个阶段。有关更多信息，请参阅资源包状态。
