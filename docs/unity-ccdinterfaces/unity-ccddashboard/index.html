Unity CCD Develop Dashboard

Important : The Unity Cloud Content Delivery documentation has moved to docs.unity.com. Please visit the new site for the most up to date information about Unity CCD.

您可以在 Develop Dashboard 中使用 Cloud Content Delivery (CCD) 来创建和维护存储桶、标示和发布版本。但是，必须先使用命令行界面 (CLI) 上传、删除和同步文件和文件夹，然后才能在 Develop Dashboard 中进行相关的管理。

例如，您可以使用 Develop Dashboard 以比 CLI 更为直观的用户界面来交互式管理应用程序的资源。

要在 Develop Dashboard 中访问 CCD，请执行以下操作：

1.如果是首次使用 Unity 服务： 1. 创建一个 Unity ID 帐户（如果还没有此帐户）。 2.从 Develop 选项卡登录到 Develop Dashboard。 3.单击 Create New Project 。在 Create New Project 窗口中，填写详细信息。 2.选择项目名称，或单击 View 。 3.在此项目的 Overview 页面左侧导航栏中，选择 Content Delivery > Buckets 。

存储桶​

在您项目的 Cloud Content Delivery 页面中，您需要创建存储桶来托管您的内容。

要添加存储桶，请执行以下操作：

1.单击 Create Bucket 。 2.在 Create Bucket 窗口中，为存储桶指定名称和可选描述。

在存储桶列表视图中，每个存储桶都有一个详细信息行，其中包含以下信息：

  • 存储桶名称
  • 存储桶的描述
  • 当前状态的快照：
    • 当前发布版本的版本号，或有关未发布更改的通知。
    • 对存储桶进行最后更改的日期。
    • 进行最新更改的人员的用户名，或自动更改的来源的名称。

您可以通过存储桶所在行末尾的 More Items 菜单 (⋮) 来编辑或删除存储桶。

要查看有关存储桶的更多信息，请在列表视图中单击存储桶的名称。在存储桶详细信息屏幕中有以下选项卡：

  • Latest ：显示存储桶的当前状态及其所有内容的最新版本。在这里，您可以创建发布版本。 资源列表是分页的，可搜索，并包含筛选条件。 如果您的存储桶没有任何未发布的更改，则可以从此选项卡推进某个发布版本。
  • Releases ：显示您为此项目创建的所有发布版本。
  • Badges ：列出此发布版本可用的所有标示。

发布版本​

您可以在 Releases 选项卡上管理为项目创建的发布版本。如果您的存储桶中有任何未发布的内容，则会在发布版本的详细信息行中看到 Create Release 按钮。

要创建发布版本，请执行以下操作：

1.从 Content Delivery Buckets 页面中选择一个包含未发布更改的存储桶。 2.在存储桶的页面中，选择 Latest 选项卡。 3.选择 Create Release 按钮。 4.在 Create Release 对话框中，输入可选的发行说明，然后单击 Next 。 5.在下一页上，选择要分配给该发布版本的标示（将这些标示从当前发布版本中移除），然后选择 Next 。 6.查看您的更改，然后选择 Submit 以完成操作，或选择 Back 以继续编辑。 Unreleased Changes 通知会更改以反映新状态。

创建发布版本后，该发布版本将显示在 Releases 选项卡的列表中，并带有创建日期和所有关联的标示。您可以通过发布版本所在行末尾的 More Items 菜单 (⋮) 来为发布版本分配标示。

当您单击列表中某个发布版本的 Release # 时，您会看到一个页面，其中包含该发布版本的详细信息以及与该发布版本相关联的资源和捆绑包列表。

在项目发布管线中的某些情况下，可能需要将发布版本从一个存储桶移至另一个存储桶，此过程称为推进。如果您的存储桶中没有任何未发布的更改，则会在发布版本的详细信息行中看到 Promote Release 按钮。

要推进发布版本，请执行以下操作：

1.在 Releases 选项卡中单击要推进的发布版本的 Release # 。 2.在发布版本摘要中，单击 Promote Release 。 3.在 Promote Release 窗口中，从 Target Bucket 下拉选单中选择要将发布版本推进到的存储桶。 4.添加任何可选的发行说明，然后选择 Next 。 5.在下一页上，选择要分配给该发布版本的标示（将这些标示从当前发布版本中移除），然后选择 Next 。 6.查看您的更改，然后选择 Promote Release 以完成操作，或选择 Back 以继续编辑。

标示​

为发布版本附加标示可使您的应用程序从该标示所分配到的任何发布版本加载内容。

要创建标示，请执行以下操作：

1.从 Content Delivery Buckets 页面中选择一个存储桶。 2.选择 Badges 选项卡。 3.选择 Add New 。 4.在 Add New Badge 窗口中，为标示指定一个唯一的名称，然后从下拉列表中选择相应的发布版本。 5.选择 Submit 。

现在，该标示会显示在 Badges 选项卡的标示列表中。您可以通过标示所在行末尾的 More Items 菜单 (⋮) 来编辑该标示的相关发布版本或删除该标示。
